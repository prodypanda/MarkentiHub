# 11 — S3 Storage Setup

PandaMarket uses **S3-compatible object storage** for product images, KYC documents, digital product files, and theme assets. This guide covers both local (MinIO) and production (S3/Scaleway) setup.

---

## Overview

### Storage Buckets

| Bucket Name | Access | Contents |
|-------------|--------|----------|
| `pd-product-images` | **Public** | Product photos, store logos, thumbnails |
| `pd-private-files` | **Private** | KYC documents (RC, CIN), digital product files |
| `pd-themes` | **Public** | Theme CSS, images, fonts |

### How Files are Accessed

- **Public buckets**: Files are served directly via the S3/MinIO URL (no authentication needed)
- **Private buckets**: Files are accessed via **presigned URLs** (temporary, time-limited links generated by the backend)

## Local Development: MinIO

MinIO is included in the `docker-compose.yml` and starts automatically.

### Access the MinIO Console

| URL | Credentials |
|-----|------------|
| `http://localhost:9001` | Username: `pd_minio_admin` / Password: `pd_minio_secret_2026` |

### Buckets are Created Automatically

The `minio-init` service in `docker-compose.yml` automatically creates the 3 buckets and sets public access on `pd-product-images` and `pd-themes`.

### Verify Buckets

1. Open `http://localhost:9001` in your browser
2. Log in with the credentials above
3. Click **Buckets** in the sidebar
4. You should see all 3 buckets listed

### Upload a Test File

1. In the MinIO console, click on `pd-product-images`
2. Click **Upload** → **Upload File**
3. Select any image file
4. After upload, click on the file and copy the URL
5. Open that URL in a new browser tab — the image should display

## Production: Using a Real S3 Provider

For production, replace MinIO with a real S3-compatible storage service.

### Option 1: OVH Object Storage (Recommended for Tunisia)

1. Go to **https://www.ovh.com** → Cloud → Public Cloud
2. Create a project
3. Go to **Object Storage** → Create a container
4. Create 3 containers:
   - `pd-product-images` (Public)
   - `pd-private-files` (Private)
   - `pd-themes` (Public)
5. Go to **Users & Roles** → Create a user with "ObjectStore operator" role
6. Note your **Access Key** and **Secret Key**

### Option 2: Scaleway Object Storage

1. Go to **https://console.scaleway.com**
2. Navigate to **Object Storage** → Create a bucket
3. Create 3 buckets with the names above
4. Go to **IAM** → **API Keys** → Create a new key
5. Note your **Access Key** and **Secret Key**

### Option 3: AWS S3

1. Go to **https://aws.amazon.com** → S3
2. Create 3 buckets
3. Set bucket policies for public access on `pd-product-images` and `pd-themes`
4. Go to **IAM** → Create a user with S3 access
5. Create access keys

### Configure .env for Production

```env
# For OVH Object Storage
PD_S3_ENDPOINT=https://s3.gra.io.cloud.ovh.net
PD_S3_REGION=gra
PD_S3_ACCESS_KEY=your_ovh_access_key
PD_S3_SECRET_KEY=your_ovh_secret_key

# For Scaleway
PD_S3_ENDPOINT=https://s3.fr-par.scw.cloud
PD_S3_REGION=fr-par
PD_S3_ACCESS_KEY=your_scaleway_access_key
PD_S3_SECRET_KEY=your_scaleway_secret_key

# For AWS S3
PD_S3_ENDPOINT=https://s3.eu-west-3.amazonaws.com
PD_S3_REGION=eu-west-3
PD_S3_ACCESS_KEY=your_aws_access_key
PD_S3_SECRET_KEY=your_aws_secret_key

# Bucket names (same for all providers)
PD_S3_BUCKET_PUBLIC=pd-product-images
PD_S3_BUCKET_PRIVATE=pd-private-files
PD_S3_BUCKET_THEMES=pd-themes
```

## Bucket Policies

### Public Bucket Policy (pd-product-images, pd-themes)

If using AWS S3 or compatible, add this bucket policy to allow public reads:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::pd-product-images/*"
        }
    ]
}
```

### Private Bucket (pd-private-files)

No public access. Files are accessed only via presigned URLs generated by the backend. No bucket policy changes needed — keep default (private).

## File Upload Limits

| File Type | Max Size | Allowed Formats |
|-----------|----------|-----------------|
| Product images | 10 MB | JPEG, PNG, WebP |
| KYC documents | 10 MB | JPEG, PNG, PDF |
| Digital products | 50 MB | Any |

These limits are configured in `backend/src/utils/constants.ts` under `FILE_CONSTRAINTS`.

## Presigned URLs

PandaMarket uses **presigned URLs** for secure file operations:

### Upload Flow
```
1. Frontend requests a presigned upload URL from the backend
   → POST /api/pd/upload/presigned-url
2. Backend generates a temporary URL (valid for 15 minutes)
3. Frontend uploads the file directly to S3 using the presigned URL
4. Frontend sends the file key back to the backend to save in the database
```

### Download Flow (Digital Products / KYC Docs)
```
1. Authorized user requests a download
   → GET /api/pd/digital/:id/download
2. Backend generates a temporary download URL (valid for 1 hour)
3. User is redirected to the presigned URL to download the file
```

## Migrating from MinIO to Production S3

When you're ready to move from MinIO (local) to a real S3 provider:

1. Update `.env` with the new S3 credentials
2. Create the 3 buckets on the new provider
3. Set public access on `pd-product-images` and `pd-themes`
4. Restart the backend: `pm2 restart pd-backend`

> 💡 To migrate existing files, use the MinIO client (`mc`) to copy:
> ```bash
> mc alias set local http://localhost:9100 pd_minio_admin pd_minio_secret_2026
> mc alias set prod https://s3.gra.io.cloud.ovh.net YOUR_KEY YOUR_SECRET
> mc mirror local/pd-product-images prod/pd-product-images
> mc mirror local/pd-private-files prod/pd-private-files
> mc mirror local/pd-themes prod/pd-themes
> ```
